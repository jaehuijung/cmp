1. 파일 네이밍
- HTML 파일명은 소문자를 사용하며, 단어 사이에는 하이픈(-)을 사용합니다.
    - 하이픈을 사용하여 단어를 구분하는 이유는 URL에서의 가독성과 표준을 만족시키기 위해서입니다.

- 파일명은 페이지의 내용을 명확히 알 수 있도록 작성합니다.
    - 예: `index.html`, `about-us.html`, `contact.html`, `product-list.html`
    
2. id 네이밍
- `id`는 특정한 한 요소를 고유하게 식별하기 위해 사용됩니다. 유일해야 하며, HTML 문서 내에서 중복될 수 없습니다.
- `id` 이름은 카멜 케이스(camelCase) 또는 하이픈(-)을 사용하는 것이 일반적입니다.
    - 일관성을 위해 프로젝트 전체에서 한 가지 스타일만 사용합니다. (보통 하이픈이 선호됩니다.)

- 이름은 고유하면서도 해당 요소의 용도를 설명할 수 있어야 합니다.
    - 예: `header`, `footer`, `mainSection`, `loginForm`, `userProfile`

3. 클래스 네이밍
- `class`는 소문자로 작성하며, 여러 단어는 하이픈(-)으로 구분합니다.
    - 소문자와 하이픈을 사용하는 것이 가독성과 유지보수를 높이는 데 도움을 줍니다.
    - 예: `navbar-item`, `form-control`, `btn-primary`

- 클래스명은 역할(role)이나 목적(purpose)을 나타내도록 작성합니다.
- 재사용 가능한 스타일을 위해 구성 요소별로 클래스를 명명하는 것이 일반적입니다.

4. 속성 네이밍 (`name`, `data-*` Custom Attributes)
- `name` 속성: 폼 데이터를 서버로 전송할 때 사용되며, 명확하고 단순한 이름을 사용합니다.
    - 예: `username`, `password`, `email`

- `data-*`: 사용자 정의 데이터 속성에서는 소문자와 하이픈(-)을 사용하여 직관성을 높입니다.
    - 예: `data-user-id`, `data-product-name`

5. 태그 구조 및 들여쓰기
- HTML은 2칸 들여쓰기가 일반적입니다.
- 태그를 중첩할 때는 코드의 가독성을 위해 적절히 들여씁니다.

6. 이미지 및 리소스 파일 네이밍
- 리소스 파일명은 소문자를 사용하며, 단어 사이에 하이픈(-)을 사용합니다.
- 가능한 한 파일의 내용을 설명하는 이름을 사용합니다.
    - 예: `logo.png`, `user-profile.jpg`, `main-banner.jpg`

7. ARIA 및 접근성 속성
- ARIA(Accessible Rich Internet Applications) 속성은 소문자와 하이픈(-)을 사용합니다.
    - 예: `aria-label`, `aria-hidden`, `role`

8. 인라인 스타일 사용 금지
- HTML에서는 CSS 스타일을 클래스로 추출하고, 인라인 스타일 사용을 지양합니다.
    - 좋은 예: `<div class="my-style"></div>`
    - 나쁜 예: `<div style="color: red; font-size: 14px;"></div>`

9. 주석 처리
- 주석은 `<!-- -->`를 사용하여 작성하며, 중요하거나 복잡한 구조만 남깁니다.
- 주석 내용은 해당 코드가 수행하는 기능을 간략히 설명합니다.
