1. 패키지 네이밍
- 패키지명은 "소문자"만 사용하며, 여러 단어는 `.`(점)으로 구분합니다.
- 일반적인 형식:
    - 도메인이 포함된 경우: `com.<회사명>.<프로젝트명>`
    - 예: `com.example.myproject`
    - 도메인이 없는 로컬 프로젝트: `org.<조직명>.<모듈명>`
    - 예: `org.myteam.utils`

- 단어 사이의 구분에는 `_`(언더스코어)를 사용하지 않습니다.
- 가능한 한 짧고, 의미 있게 사용합니다.

2. 클래스 네이밍
- 클래스명은 파스칼 케이스(PascalCase)를 사용합니다.
    - 첫 번째 글자는 항상 대문자로 시작하며, 각 단어의 첫 글자도 대문자로 표기합니다.

- 클래스명은 명사형으로 짧고 의미 있게 작성합니다.
    - 예: `Customer`, `OrderProcessor`, `PaymentGateway`

- 클래스 파일명은 클래스 이름과 반드시 일치해야 합니다.

3. 인터페이스 네이밍
- 인터페이스명은 PascalCase를 사용하고, 일반적으로 동작/행동을 설명하는 단어(Suffix: -able, -ible 등)로 끝납니다.
- 예: `Readable`, `Runnable`, `Cloneable`
- 인터페이스가 특정 도메인 클래스를 설명한다면 명사형도 허용됩니다.

4. 메서드 네이밍
- 메서드명은 카멜 케이스(camelCase)를 사용합니다.
    - 첫 단어는 소문자로 작성하고, 두 번째 단어부터는 대문자로 시작합니다.

- 메서드명은 동사를 사용하거나 명령형으로 작성하는 것이 일반적입니다.
    - 예: `getName()`, `calculateTotalPrice()`, `fetchRecords()`

- Boolean 값을 반환하는 메서드는 `is`, `has` prefix를 사용합니다.
    - 예: `isValid()`, `hasPermission()`

5. 변수 네이밍
- 변수명은 camelCase를 사용하며, 의미 있는 이름을 부여합니다.
    - 첫 번째 단어는 소문자로 시작하고, 단어의 경계를 대문자로 구분합니다.
    - 일반적으로 명사나 명사구를 사용합니다.

- 약어 사용은 지양하며, 이름은 충분히 의미를 담아야 합니다.
    - 예: `customerName`, `totalAmount`, `itemList`

6. 상수 네이밍
- 상수는 모두 대문자(UPPERCASE)로 작성하며, 단어는 `_`(언더스코어)로 구분합니다.
- 클래스 내 `static final`로 선언된 변수에 주로 사용됩니다.
- 항상 명확하고 구체적인 의미를 담습니다.
    - 예: `MAX_USERS`, `DEFAULT_TIMEOUT`, `PI`

7. 열거형(Enum) 네이밍
- 열거형 클래스명은 PascalCase를 사용합니다.
- 열거형 내부 값은 모두 UPPERCASE를 사용합니다.

8. 파일 네이밍
- 파일명은 클래스명 또는 인터페이스명과 동일하게 작성해야 합니다.
    - 파일명 예: `Customer.java`, `OrderProcessor.java`
